# 🚀 Developer Portfolio — Retro HUD Interface

A modular, state-driven single-page portfolio built entirely with **vanilla JavaScript, HTML, and CSS**.

This project focuses on structured architecture, dynamic rendering, and performance-conscious UI design — without relying on frontend frameworks.

---

## ✨ Overview

This portfolio was built to explore:

- Configuration-driven UI rendering  
- State-managed view transitions  
- Modular JavaScript architecture  
- Clean separation of structure and logic  
- Performance-optimized visual effects  

Rather than using a framework, the goal was to deeply understand core web fundamentals and build a scalable interface from scratch.

---

## 🧠 Architecture Philosophy

The application follows a modular, state-driven approach:

- Centralized state object controlling view switching and module updates  
- Proxy-based state triggers to re-render specific UI sections  
- Template cloning with dynamic DOM rendering  
- Clear separation of concerns across modules  
- No external UI libraries  

Each module is isolated and responsible for a specific domain:

- Navigation  
- Projects  
- Project Details  
- Skills  
- About  
- Contact  
- Background Animation

## 💡 Why Vanilla JS?

The goal of this project was not to avoid frameworks — but to understand the fundamentals behind them:

- Rendering flow  
- State updates  
- Event delegation  
- Modular composition  
- DOM manipulation patterns  

By implementing these concepts manually, the architectural foundations become clearer and transferable to frameworks like React or Angular.
